# Inbox

사용자의 수신함. 아이디어, 피드백, 메모를 자유롭게 적는 공간.

---

## 처리 지침

**참고**: 이 공간은 계획이나 약속을 정하는 곳이지, 코드 작성이 아닙니다.

사용자가 "수신함 검토해줘" 요청 시:

1. 아래 내용 전체 읽기
2. 항목별로 재정리:
   - 목록 형식으로 변환
   - 첫 번째 depth: Task `- [ ]` 사용
   - 두 번째 depth 이후: `-` 만 사용해서 보조 설명 추가
3. 각 항목의 성격에 따라 적절한 문서에 반영:
   - 프로젝트 전체 지침 → `/CLAUDE.md` `/AGENT.md` `/GEMINI.md` 동일하게
   - 프로젝트 방향/로드맵 → `plan.md`
   - 확정된 규칙/스펙 → `specs/주제.md`
   - 결정 히스토리 → `decisions/YYMMDD-주제.md`
   - stage 관련 → `stages/stageN_제목.md`
   - 작업 기록 → `devlog/YYMMDD.md`
   - AI 협업 규칙 → `GUIDE.md`
4. 반영 결과를 사용자에게 보고
5. **사용자가 직접 inbox를 비움**:
   - 조치하고 완료된 항목: `- [x]` 로 표시하고 아래로 이동
   - 아직 처리되지 않거나 애매모호한 항목: 그대로 남겨둠
   - 최종 검토 후 완료된 항목은 사용자가 삭제
   - AI가 직접 삭제하지 않음

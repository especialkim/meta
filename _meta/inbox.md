# Inbox

사용자의 수신함. 아이디어, 피드백, 메모를 자유롭게 적는 공간.

---

## 처리 지침

사용자가 "수신함 검토해줘" 요청 시:

1. 아래 내용 전체 읽기
2. 각 항목의 성격에 따라 적절한 문서에 반영:
   - 프로젝트 전체 지침 → `/CLAUDE.md`
   - 프로젝트 방향/로드맵 → `plan.md`
   - 확정된 규칙/스펙 → `specs/주제.md`
   - 결정 히스토리 → `decisions/YYMMDD-주제.md`
   - stage 관련 → `stages/stageN_제목.md`
   - 작업 기록 → `devlog/YYMMDD.md`
   - AI 협업 규칙 → `GUIDE.md`
3. 반영 결과를 사용자에게 보고
4. **사용자가 직접 inbox를 비움** (AI가 삭제하지 않음)
